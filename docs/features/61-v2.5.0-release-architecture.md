## 1. ENH-08: Expanded Error Codes

### Current State
```go
type AppError struct {
    Code    int    // HTTP status code (e.g., 404, 500)
    Message string
    Err     error
}
```
`Code` is the HTTP status but is named ambiguously. No field for machine-readable error identifiers like `"VALIDATION_FAILED"`.

### Design

Rename the field to `Status`, add `Code string`, and provide a backward-compatible deprecated getter:

```go
type AppError struct {
    Status  int    // HTTP status code (e.g., 404, 500)
    Code    string // Machine-readable error code (e.g., "NOT_FOUND", "VALIDATION_FAILED")
    Message string // User-facing safe message
    Err     error  // Internal error (logged, never exposed in production)
}

// HTTPStatus returns the HTTP status code.
// Deprecated: Access .Status directly instead.
func (e *AppError) HTTPStatus() int {
    return e.Status
}
```

Update all 7 factory functions to set both `Status` and a default `Code`:

```go
func NotFound(message string) *AppError {
    return &AppError{Status: 404, Code: "NOT_FOUND", Message: message}
}
func BadRequest(message string) *AppError {
    return &AppError{Status: 400, Code: "BAD_REQUEST", Message: message}
}
func Internal(err error) *AppError {
    return &AppError{Status: 500, Code: "INTERNAL_ERROR", Message: "internal server error", Err: err}
}
func Unauthorized(message string) *AppError {
    return &AppError{Status: 401, Code: "UNAUTHORIZED", Message: message}
}
func Forbidden(message string) *AppError {
    return &AppError{Status: 403, Code: "FORBIDDEN", Message: message}
}
func Conflict(message string) *AppError {
    return &AppError{Status: 409, Code: "CONFLICT", Message: message}
}
func Unprocessable(message string) *AppError {
    return &AppError{Status: 422, Code: "UNPROCESSABLE", Message: message}
}
```

Add `WithCode()` builder for custom codes:
```go
func (e *AppError) WithCode(code string) *AppError {
    e.Code = code
    return e
}
```

Usage: `errors.BadRequest("invalid email").WithCode("INVALID_EMAIL")`

Update `ErrorResponse()` to include code:
```go
func (e *AppError) ErrorResponse() map[string]any {
    resp := map[string]any{
        "success": false,
        "error":   e.Message,
        "code":    e.Code,
    }
    if config.IsDebug() && e.Err != nil {
        resp["internal"] = e.Err.Error()
    }
    return resp
}
```

**Breaking change mitigation**: The field rename (`Code int` в†’ `Status int`) is breaking for anyone accessing `err.Code` and expecting an int. Since `Code` is now a string, existing code like `if err.Code == 404` won't compile. This is intentional вЂ” caught at compile time, easy to fix. The migration path: `err.Code` в†’ `err.Status`.

Update all internal references in the framework (middleware error handler, etc.).

## 2. ENH-09: Rate Limit Convenience Helpers

### Current State
`RateLimitWithConfig` accepts a custom `KeyFunc`. Users must write their own key functions every time.

### Design

Add pre-built key functions and a `ParseRate()` helper:

```go
// core/middleware/ratelimit.go вЂ” new helpers

// KeyByIP returns a KeyFunc that uses the client IP (default behavior, explicit).
func KeyByIP() func(c *gin.Context) string {
    return func(c *gin.Context) string {
        return c.ClientIP()
    }
}

// KeyByUserID returns a KeyFunc that uses a value from the Gin context.
// Typically set by auth middleware (e.g., c.Set("userID", id)).
func KeyByUserID(contextKey string) func(c *gin.Context) string {
    return func(c *gin.Context) string {
        if v, exists := c.Get(contextKey); exists {
            return fmt.Sprintf("user:%v", v)
        }
        return c.ClientIP() // fallback to IP if no user
    }
}

// KeyByHeader returns a KeyFunc that uses a specific request header value.
func KeyByHeader(header string) func(c *gin.Context) string {
    return func(c *gin.Context) string {
        if v := c.GetHeader(header); v != "" {
            return fmt.Sprintf("header:%s:%s", header, v)
        }
        return c.ClientIP()
    }
}

// KeyByBodyField returns a KeyFunc that reads a JSON body field.
// Note: This reads and re-buffers the request body.
func KeyByBodyField(field string) func(c *gin.Context) string {
    return func(c *gin.Context) string {
        var body map[string]any
        if err := c.ShouldBindJSON(&body); err == nil {
            if v, ok := body[field]; ok {
                return fmt.Sprintf("body:%s:%v", field, v)
            }
        }
        return c.ClientIP()
    }
}

// ParseRate parses a rate string in ulule format (e.g., "100-M", "10-S", "1000-H").
// Returns an error if the format is invalid. Useful for validating rate strings
// before passing them to RateLimitWithConfig.
func ParseRate(rate string) (limiter.Rate, error) {
    return limiter.NewRateFromFormatted(rate)
}
```

Usage:
```go
// Per-user rate limit: 100 requests per minute
r.Use(middleware.RateLimitWithConfig(middleware.RateLimitConfig{
    Rate:    "100-M",
    KeyFunc: middleware.KeyByUserID("userID"),
}))

// Per-API-key rate limit
r.Use(middleware.RateLimitWithConfig(middleware.RateLimitConfig{
    Rate:    "1000-H",
    KeyFunc: middleware.KeyByHeader("X-API-Key"),
}))
```

## 3. IMP-03: Seeder Zero-Registered Warning

### Current State
`db:seed` has `seederFn == nil` check and returns error. But if `SetSeeder()` was called with a function that has no seeders registered, it runs silently with no output.

### Design

This is a documentation/pattern issue more than a code issue. The `seederFn` already returns an error when nil. The improvement is: if `seederFn` succeeds but produced no output, log a hint:

```go
// core/cli/seed.go вЂ” after seederFn() call
fmt.Fprintln(cmd.OutOrStdout(), "Database seeding complete.")
fmt.Fprintln(cmd.OutOrStdout(), "Tip: If no seeders ran, ensure they are registered in your seeders package init().")
```

Actually, the better approach: let the seeder function in the starter indicate how many seeders ran. Keep framework side simple вЂ” just improve the error message when `seederFn` is nil.

## 4. IMP-04: `db:seed --list` Flag

### Current State
No way to list available seeders.

### Design

This requires the `SeederFunc` to support a list mode. We can't enumerate seeders from the framework since they're registered in user code. The cleanest approach:

Add a `--list` flag. When set, call `seederFn` with a special sentinel name `"__list"`. The starter's seeder function recognizes this and prints names.

Simpler alternative: Add `SeederListFunc` hook:

```go
// core/cli/hooks.go
type SeederListFunc func() []string
var seederListFn SeederListFunc
func SetSeederList(fn SeederListFunc) { seederListFn = fn }
```

```go
// core/cli/seed.go
if list, _ := cmd.Flags().GetBool("list"); list {
    if seederListFn == nil {
        fmt.Fprintln(cmd.OutOrStdout(), "No seeder list registered. Call cli.SetSeederList() to enable.")
        return nil
    }
    for _, name := range seederListFn() {
        fmt.Fprintln(cmd.OutOrStdout(), "  - "+name)
    }
    return nil
}
```

## 6. IMP-06: `container.TryMake()`

### Current State
`Make()` panics if service not found. `Has()` exists but requires a separate check.

### Design

```go
// core/container/container.go
func (c *Container) TryMake(name string) (interface{}, error) {
    c.mu.RLock()
    if inst, ok := c.instances[name]; ok {
        c.mu.RUnlock()
        return inst, nil
    }
    factory, ok := c.bindings[name]
    c.mu.RUnlock()
    if !ok {
        return nil, fmt.Errorf("service not found: %s", name)
    }
    return factory(c), nil
}

// TryMake resolves a service with type safety, returning an error instead of panicking.
func TryMake[T any](c *Container, name string) (T, error) {
    var zero T
    raw, err := c.TryMake(name)
    if err != nil {
        return zero, err
    }
    val, ok := raw.(T)
    if !ok {
        return zero, fmt.Errorf("service %s: type assertion failed", name)
    }
    return val, nil
}
```

## 8. FEAT-02: `db:wipe` Command

### Design

Truncate all user tables (not migration tracking). Safety checks like `migrate:fresh`.

```go
// core/cli/db_wipe.go
var dbWipeCmd = &cobra.Command{
    Use:   "db:wipe",
    Short: "Truncate all database tables",
    RunE: func(cmd *cobra.Command, args []string) error {
        if config.AppEnv() == "production" {
            force, _ := cmd.Flags().GetBool("force")
            if !force {
                return fmt.Errorf("refusing to wipe database in production without --force")
            }
        }
        application := NewApp(service.ModeAll)
        db := container.MustMake[*gorm.DB](application.Container, "db")

        tables, _ := db.Migrator().GetTables()
        for _, table := range tables {
            if table == "migrations" {
                continue // preserve migration tracking
            }
            // Use parameterized GORM Exec to avoid SQL injection
            // (table names from GetTables are trusted DB metadata)
            switch db.Dialector.Name() {
            case "postgres":
                db.Exec(fmt.Sprintf("TRUNCATE TABLE %q CASCADE", table))
            case "mysql":
                db.Exec(fmt.Sprintf("TRUNCATE TABLE `%s`", table))
            default: // sqlite
                db.Exec(fmt.Sprintf("DELETE FROM \"%s\"", table))
            }
        }
        fmt.Fprintln(cmd.OutOrStdout(), "All tables truncated.")
        return nil
    },
}
```

Note: `TRUNCATE ... CASCADE` is PostgreSQL-specific. For MySQL use `TRUNCATE TABLE`, for SQLite use `DELETE FROM`. We should use GORM's dialect detection or a simple switch.

## 10. FEAT-06: Configurable Startup Banner

### Current State
Banner is hardcoded in `serve` command's `RunE`.

### Design

Extract banner to a function with env override:

```go
// core/cli/serve.go
func printBanner(appName, appEnv, mode, port string) {
    banner := config.Env("APP_BANNER", "")
    if banner != "" {
        fmt.Println(banner)
        return
    }
    fmt.Println("=================================")
    fmt.Printf("  %s Framework\n", appName)
    fmt.Println("  github.com/raiworks/rapidgo")
    fmt.Println("=================================")
    fmt.Printf("  Environment: %s\n", appEnv)
    fmt.Printf("  Mode: %s\n", mode)
    fmt.Printf("  Port: %s\n", port)
    fmt.Printf("  Debug: %v\n", config.IsDebug())
    fmt.Println("=================================")
}
```

If `APP_BANNER` env var is set, it prints that instead of the default. Otherwise unchanged.
