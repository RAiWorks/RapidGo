# 💬 Discussion: v2.4.0 Release

## 1. What Are We Building?

A minor release (v2.4.0) fixing 2 critical bugs and shipping 4 quick-win improvements to the RapidGo framework. All changes are backward-compatible.

## 2. Why?

Two production deployments (RAiSites and SocioSky) surfaced these issues. BUG-02 is a ship-blocker — `SetRoutes()` silently does nothing in single-port mode, which is the default for most projects.

## 3. Origin

Items cataloged in `docs/enhancements-improvements-bugs/index.md`. Full cross-check against codebase documented in `03-rapidgo-next-discussion.md`.

## 4. Scope

### Bug Fixes
- **BUG-02**: `serveSingle()` never calls `routeRegistrar` — routes registered via `SetRoutes()` are silently ignored in single-port mode
- **BUG-03**: `serveMulti()` creates new routers with `router.New()`, discarding all middleware/routes that providers registered on the container's router

### Improvements
- **IMP-01**: Server timeouts (read, write, idle, shutdown) are hardcoded — make them configurable via env vars
- **IMP-02**: `config.Load()` is called in `serve`, `work`, `schedule:run` but NOT in `migrate`, `migrate:rollback`, `migrate:status`, `db:seed` — these commands can't read `.env`

### Features
- **FEAT-04**: Router `NoRoute()` method — currently requires `r.Engine().NoRoute()` which breaks encapsulation
- **FEAT-03**: Health endpoint returns `{"status":"ok"}` only — add app version for debugging

### Out of Scope
- Everything in v2.5.0+ (error codes, rate limit helpers, DX improvements)
- No new dependencies

## 5. Dependencies & Risk

| Risk | Mitigation |
|---|---|
| BUG-02 fix changes `serveSingle()` behavior | Routes that previously silently didn't register will now register — this is correct behavior, not a regression |
| BUG-03 fix changes `serveMulti()` to use container router | Provider-registered middleware now applies — this is the intended behavior |
| IMP-01 adds env vars | All have sensible defaults matching current hardcoded values |

All items verified against codebase. No open questions. See `03-rapidgo-next-discussion.md` for full cross-check.
