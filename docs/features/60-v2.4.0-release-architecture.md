## 1. BUG-02: `serveSingle()` Skips `routeRegistrar`

### Current State
```go
// core/cli/serve.go — serveSingle()
func serveSingle(application *app.App, mode service.Mode) error {
    port := resolvePort(mode)
    r := container.MustMake[*router.Router](application.Container, "router")
    return server.ListenAndServe(server.Config{
        Addr:    ":" + port,
        Handler: r,
        // ...
    })
}
```
It grabs the router from the container and serves it directly. **Never calls `routeRegistrar`** — so `SetRoutes()` does nothing in single-port mode (which is the default).

### Fix

Call `applyRoutesForMode()` on the container's router before serving, exactly like `serveMulti()` does but using the existing router instead of creating a new one:

```go
func serveSingle(application *app.App, mode service.Mode) error {
    port := resolvePort(mode)
    r := container.MustMake[*router.Router](application.Container, "router")
    applyRoutesForMode(r, application.Container, mode)
    return server.ListenAndServe(server.Config{
        Addr:    ":" + port,
        Handler: r,
        // ...
    })
}
```

This ensures `routeRegistrar`, template loading, static file serving, and health routes all work in single-port mode.

## 2. BUG-03: `serveMulti()` Discards Provider Work

### Current State
```go
// core/cli/serve.go — serveMulti()
for _, svc := range mode.Services() {
    r := router.New()  // <-- creates brand new router
    applyRoutesForMode(r, application.Container, svc)
    // ...
}
```
Creates new routers, discarding middleware/routes that providers registered on the container's router during `Boot()`.

### Fix

For multi-port mode, we need separate routers per service (correct), but each should inherit the **global middleware** from the container's router. The approach:

1. Keep creating per-service routers (necessary for separate ports)
2. Copy global middleware from the container's router to each per-service router
3. Add a `Router.GlobalMiddleware()` getter and `Router.UseGlobal()` setter

```go
// core/router/router.go — new methods
func (r *Router) GlobalHandlers() []gin.HandlerFunc {
    return r.engine.Handlers
}
```

```go
// core/cli/serve.go — serveMulti() fix
containerRouter := container.MustMake[*router.Router](application.Container, "router")

for _, svc := range mode.Services() {
    r := router.New()
    // Copy global middleware from the container's router
    for _, h := range containerRouter.GlobalHandlers() {
        r.Use(h)
    }
    applyRoutesForMode(r, application.Container, svc)
    // ...
}
```

## 3. IMP-01: Configurable Server Timeouts

### Current State
All four timeouts are hardcoded literals:
```go
ReadTimeout:     15 * time.Second,
WriteTimeout:    15 * time.Second,
IdleTimeout:     60 * time.Second,
ShutdownTimeout: 30 * time.Second,
```

### Design

Add a helper that reads timeout env vars with fallback defaults:

```go
// core/cli/serve.go — new helper
func resolveServerTimeouts() (read, write, idle, shutdown time.Duration) {
    read = parseDuration(config.Env("SERVER_READ_TIMEOUT", "15s"), 15*time.Second)
    write = parseDuration(config.Env("SERVER_WRITE_TIMEOUT", "15s"), 15*time.Second)
    idle = parseDuration(config.Env("SERVER_IDLE_TIMEOUT", "60s"), 60*time.Second)
    shutdown = parseDuration(config.Env("SERVER_SHUTDOWN_TIMEOUT", "30s"), 30*time.Second)
    return
}

func parseDuration(s string, fallback time.Duration) time.Duration {
    d, err := time.ParseDuration(s)
    if err != nil {
        return fallback
    }
    return d
}
```

Both `serveSingle()` and `serveMulti()` use this instead of hardcoded values.

**Env vars**: `SERVER_READ_TIMEOUT`, `SERVER_WRITE_TIMEOUT`, `SERVER_IDLE_TIMEOUT`, `SERVER_SHUTDOWN_TIMEOUT`. Format: Go duration strings (`15s`, `1m`, `500ms`).

## 4. IMP-02: Consistent `config.Load()` Across CLI Commands

### Current State
| Command | Calls `config.Load()`? |
|---------|----------------------|
| `serve` | ✅ Yes |
| `work` | ✅ Yes |
| `schedule:run` | ✅ Yes |
| `migrate` | ❌ No |
| `migrate:rollback` | ❌ No |
| `migrate:status` | ❌ No |
| `db:seed` | ❌ No |

### Design

Move `config.Load()` into `NewApp()` so every command that bootstraps the application automatically loads `.env`:

```go
// core/cli/root.go — NewApp()
func NewApp(mode service.Mode) *app.App {
    config.Load()  // <-- add here

    application := app.New()
    if bootstrapFn != nil {
        bootstrapFn(application, mode)
    }
    application.Boot()
    return application
}
```

Remove the explicit `config.Load()` calls from `serve`, `work`, and `schedule:run` RunE functions. The `serve` command currently calls `config.Load()` *before* `NewApp()` to read `RAPIDGO_MODE` — we need to keep that early call but make `NewApp()` call it too (idempotent, `godotenv.Load()` is safe to call multiple times).

## 5. FEAT-04: Router `NoRoute()` Method

### Current State
No `NoRoute()` on Router. Users must call `r.Engine().NoRoute()` directly.

// core/router/router.go — new method
func (r *Router) NoRoute(handlers ...gin.HandlerFunc) {

## 6. FEAT-03: Health Endpoint with Version

### Current State
```go
c.JSON(http.StatusOK, gin.H{"status": "ok"})
```

### Design

Accept an optional version string:

```go
// core/health/health.go — updated signature
func Routes(r *router.Router, dbFn func() *gorm.DB, version ...string) {
    ver := ""
    if len(version) > 0 {
        ver = version[0]
    }
    r.Get("/health", func(c *gin.Context) {
        resp := gin.H{"status": "ok"}
        if ver != "" {
            resp["version"] = ver
        }
        c.JSON(http.StatusOK, resp)
    })
    // readiness endpoint unchanged
}
```

Using variadic `version ...string` preserves backward compatibility — existing calls with 2 args still work. The starter and website can pass `cli.Version` or their own version.
