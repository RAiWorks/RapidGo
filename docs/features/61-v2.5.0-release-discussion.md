# 💬 Discussion: v2.5.0 Release

> **Feature**: `61` — v2.5.0 Release
> **Date**: 2026-03-15
> **Status**: COMPLETE

---

## 1. What Are We Building?

A minor release (v2.5.0) delivering DX improvements, an expanded error system, rate limit convenience helpers, new CLI commands, and container safety. 10 items total — all backward-compatible.

## 2. Why?

Production experience from RAiSites and SocioSky exposed a pattern: the framework has good foundations but lacks polish in error responses, CLI tooling, and developer-facing conveniences. These are all additive improvements that make the day-to-day development experience smoother.

## 3. Origin

Items cataloged in `docs/enhancements-improvements-bugs/index.md`. Full cross-check in `03-rapidgo-next-discussion.md`.

## 4. Scope

### Enhancements
- **ENH-08**: Expanded error codes — rename `AppError.Code int` → `Status int`, add `Code string` for machine-readable error identifiers. Provide deprecated `Code int` getter during v2.x.
- **ENH-09**: Rate limit convenience helpers — `KeyByIP()`, `KeyByUserID()`, `KeyByBodyField()`, `KeyByHeader()`, and `ParseRate()` to complement existing `RateLimitWithConfig` API.

### Improvements
- **IMP-03**: Seeder zero-registered warning — log a warning when `db:seed` runs but no seeders are registered (blank import missed)
- **IMP-04**: `db:seed --list` flag — list available seeders without running them
- **IMP-05**: `migrate:fresh` command — drop all tables and re-run migrations
- **IMP-06**: `container.TryMake()` — safe alternative to `Make()` that returns `(value, error)` instead of panicking

### Features
- **FEAT-01**: `APP_ENV=local` graceful skip — providers can check env and skip initializing non-essential services (e.g., mail, queue) in local development
- **FEAT-02**: `db:wipe` truncate command — truncate all user tables (preserving schema)
- **FEAT-05**: `make:seeder` scaffold command — generate seeder boilerplate
- **FEAT-06**: Configurable startup banner — allow customization via env var

### Out of Scope
- Logger interface (v2.6.0)
- Typed config `LoadConfig[T]()` (v2.6.0)
- JWT `GenerateTokenFromString()` (v2.6.0)
- No new external dependencies

## 5. Dependencies & Risk

| Risk | Mitigation |
|---|---|
| ENH-08 renames `AppError.Code` field | Provide deprecated `Code` getter. Existing `err.Code` access works during v2.x via method. |
| `migrate:fresh` drops all tables | Require `--force` flag in production. Log warning otherwise. |
| `db:wipe` truncates data | Refuse to run unless `APP_ENV != production` or `--force` is passed |
