# 🔍 Review: v2.7.0 Release

> **Feature**: `63` — v2.7.0 Release

## Summary

Shipped 2 enhancements in a single build session: `UUIDBaseModel` for UUID primary keys and `make:module` domain scaffold command. 12 files changed, 647 lines added. No breaking changes. No new dependencies. ENH-03 (advanced auth) deferred to `rapidgo-modules` lifecycle.

## What Went Well

- **Scope reduction was correct** — original v2.7.0 plan included ENH-03 (advanced auth) + MOD-01 (authstore), which are 5+ days of work in a separate repo. Descoping to 2 framework items kept the release tight and shippable.
- **`UUIDBaseModel` BeforeCreate hook** — clean GORM integration. Auto-generates UUID v4 on create, preserves manually-set IDs. 5 tests all pass on first run.
- **`make:module` template approach** — reused the proven `scaffold()` pattern from existing commands. 4 files generated with proper package naming via `{{.Package}}` (snake_case).
- **Cross-check was perfectly clean** — 0 discrepancies between architecture and implementation.
- **UUID dependency already existed** — `github.com/google/uuid` was already in `go.mod`, no dependency additions needed.

## What Could Be Improved

- **`make:module` creates models with `BaseModel` only** — template hardcodes `models.BaseModel`. Could offer a `--uuid` flag to use `UUIDBaseModel` instead. Low priority — users can edit the generated file.
- **No migration template in module** — `make:module` creates `models.go`, `service.go`, `controller.go`, `routes.go` but no migration file. Users must create migrations separately. Could add a `migrations/` subdirectory with a starter migration.
- **ENH-03 scope unclear** — advanced auth was deferred but the tracker originally combined it with ENH-02 and ENH-07 under v2.7.0. The index.md description was stale until updated during ship. Should have updated it during discussion.

## Decisions Made

| Decision | Rationale |
|----------|-----------|
| `ID string` over `uuid.UUID` type | String is simpler for JSON serialization, GORM compatibility, and cross-dialect support (char(36) works everywhere) |
| `BeforeCreate` hook for auto-generation | Standard GORM pattern. Runs automatically on `db.Create()`. Can be overridden by setting ID before create. |
| 4 files per module (no migration) | Matches the most common domain pattern. Migrations are project-specific — better created manually or via `make:migration`. |
| Defer ENH-03 + MOD-01 | Advanced auth is a separate concern that lives in `rapidgo-modules`, not the core framework. Needs its own lifecycle. |
